-- Library API root set for the kimchi package.
--
-- The curated top-level surface the library exists to provide. Lean has no export-list
-- mechanism (everything non-`private` is visible once imported), so this manifest — not the
-- type system — defines the public API. It is the root set for dead-code analysis:
--
--   a declaration in `Kimchi.*` is LIVE iff it is in the transitive constant-dependency
--   closure of these roots; anything compiled but outside that closure is dead.
--
-- The list is a MINIMAL GENERATING SET: a result reachable from a listed root is deliberately
-- not listed (e.g. `Kimchi.Protocol.sound` and the openings seam
-- `kimchiProof_sound_of_openings` are live through every run-level root; `zH_dvd_iff` and
-- `dvd_separation` through the whole soundness line). `scripts/check_axioms.lean` separately
-- pins the existence and axiom closure of the headline theorems, including ones unlisted here.
--
-- Format: one fully-qualified declaration name per line; `--` lines and blanks are ignored.
--
-- Note: the `kimchi-demo` executable (Main.lean) is a SEPARATE build root, reaching the demo
-- defs `Kimchi.Gate.{egQ, egGood, egBad, satisfies}`. Those are demo-only and are not part
-- of this library API surface, so they are reachable through the executable rather than here.

-- The standard-model run-level capstones (grid path): a real accepted run's own sponge
-- challenges over the run's own commitments — unless the run's (β, γ, α, ζ) land in the four
-- bounded bad sets, the index is satisfiable by a witness table fixed before the challenges.
-- The special-soundness grid (accepting deployed Ipa.verify batch openings at
-- injectively-varied ξ/r) is a HYPOTHESIS; each node's transcript tree is derived from
-- poseidon_fiat_shamir_{vesta,pallas}. The quotient residue (t, hdeg, heq) is the one explicit
-- hypothesis beyond the grids. The whole reduction stack these compose — kimchiProof_sound,
-- the openings seam `kimchiProof_sound_of_openings`, and the PCS-free protocol soundness
-- `Kimchi.Protocol.sound` it bottoms out in — is live via reachability, not listed.

-- The algebraic quotient (AGM reading): an algebraic prover supplies SRS-basis representations
-- of its committed rows AND the 7 `tComm`-chunk representations, so a SINGLE accepted opening
-- suffices (no grid, no density) and the quotient `t` — the genuine degree-`< 7n` assembly
-- `ftChunkAssembly` of the committed chunks — and the Maller/ft identity are DERIVED from a
-- checked ft opening via `ft_identity_of_chunks`: no residue hypothesis. The satisfying table
-- is named explicitly (the representations' own data). The FiatShamirTreeB tree stays a
-- caller-supplied hypothesis (not derived from an Ipa.verify acceptance), so the axiom set has
-- NO poseidon_fiat_shamir. The residue-carrying `kimchiProof_sound_algebraic` it consumes is
-- live via reachability, not listed.

-- THE TERMINAL THEOREM — AGM soundness of the deployed kimchi verifier. From a genuine
-- `verify = true`, for an algebraic prover (SRS-basis representations of the run's 45 batch rows

-- the index — the ft/quotient residue DISSOLVED (no `t`/`hteq`), the grid BYPASSED (fed the
